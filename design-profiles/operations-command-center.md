# Operations Command Center

Design for speed, clarity, and high information throughput.

## Core Direction

- The primary surface is a live workbench, not a marketing page.
- Users should see actionable queue state immediately.
- The layout should feel competent and operational, not playful.

## Layout

- Prefer a left navigation rail plus a wide content pane.
- Use persistent filter bars above lists and queues.
- Keep summaries close to the work surface: KPI strip on top, queue below, detail drawer or side panel adjacent.
- Avoid oversized page chrome. Content density matters.

## Visual Tone

- Neutral base palette with one assertive accent.
- Backgrounds should be flat and quiet.
- Typography should do most of the hierarchy work.
- Borders and row separators are better than decorative cards for large data regions.

## Data Surfaces

- Tables and queues should be scan-friendly first, beautiful second.
- Status should be visible in shape plus color, not color alone.
- Counts, timestamps, owners, and next actions should be visible without opening a detail page.
- Empty states should explain what to do next, not just state that nothing exists.

## Interaction

- Filters, bulk actions, and sorting should be obvious.
- Row click behavior must be predictable.
- Critical actions should stay near the relevant row or detail pane.
- Long forms should be sectioned with sticky local navigation when possible.

## Avoid

- Hero sections
- full-width decorative banners
- giant cards around every section
- excessive whitespace that hides operational state
- rainbow status systems
