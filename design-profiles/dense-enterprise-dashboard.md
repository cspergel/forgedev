# Dense Enterprise Dashboard

Design for serious B2B software where users live in the product all day.

## Core Direction

- Default to clarity, density, and composure.
- A mature dashboard should feel expensive because it is disciplined, not because it is ornamental.
- The visual system should support deep navigation without becoming visually noisy.

## Layout

- Use a stable shell: sidebar, top utility bar, primary content region.
- Secondary navigation can live as tabs, segmented controls, or subnav rows inside the content pane.
- Prefer split views and drawers over route thrash when users compare lists and details frequently.

## Information Density

- Fit meaningful information above the fold.
- Lists should carry enough metadata to reduce unnecessary clicks.
- Reporting pages can use cards, but cards should hold actual metrics, not just section titles.
- Settings pages should group controls by domain with short explanatory copy.

## Visual Language

- Use neutral surfaces and one accent family.
- Small radius, modest shadows, strong alignment.
- Grid rhythm matters more than flourish.
- Charts should be clear and quiet; avoid decorative gradients or stacked gimmicks unless analytically justified.

## Interaction

- Search, filter, sort, and scope controls should always be easy to find.
- Bulk actions should feel safe and reviewable.
- Destructive actions must be visually distinct and confirmation-gated.
- Loading states should preserve layout stability.

## Avoid

- oversized hero headers
- huge empty gutters
- glossy startup-marketing visuals in core app surfaces
- every section rendered as its own isolated card island
- mismatched patterns between tables, forms, and settings pages
